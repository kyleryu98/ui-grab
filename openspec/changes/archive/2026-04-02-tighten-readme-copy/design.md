# Design

The README should lead with the product description, package names, install commands, and the shortest path to first use. The policy behind language support does not need a visible banner because the repository now only ships the English source file.

The MCP bridge explanation should stay, but in a tighter form that answers one question quickly: when `ui-grab-mcp` is needed and when it is not.

`packages/grab/README.md` should continue matching the root `README.md` because the package build script copies the root README into the published package.
